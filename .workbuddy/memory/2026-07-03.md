用户提供了 UI 设计稿（006.png）要求审查 HexImg（Go + Fyne 图片格式转换工具，路径 D:\Github\HexImg）的 UI 颜值与代码 bug，要求不改文件、仅读取分析。

### 发现的问题

**致命级：**
1. 浅色模式半成品 — `cmd/heximg/main.go` 多处硬编码 `ui.XxxColor(true)`（深色）：
   - 第190行标题 `canvas.NewText("HexImg", ui.TextColor(true))`
   - 第193行 subtitle `ui.MutedTextColor(true)`
   - 第238行 `fluentCard` 内 title `ui.TextColor(true)`
   - 第243行卡片背景 `ui.CardColor(true)`
   `fluentCard()` 不接受 darkMode 参数也无刷新机制，切主题后卡片仍为深灰块。
2. 中文 Windows 路径乱码 — `native_windows.go` PowerShell 设 UTF-8 输出，但 Go `cmd.Output()` 用系统 OEM(GBK) 解码，中文路径必乱码。

**中级：**
3. PNG 质量 slider 语义反转 — `pngCompression()` quality=100→compression=0(最大文件)，误导用户。
4. 日志截断按 byte 切 UTF-8 会切坏中文字符（`main.go` 第342行）。
5. macOS `CFBundleShortVersionString` 带 `v` 前缀，不符 Apple 规范。

**CI 致命错误（用户第二轮指出，我首轮漏检）：**
6. `.github/workflows/release.yml` 第50行 windows-arm64 matrix：
   `mingw-w64-clang-aarch64-winpthread`（单数）→ MSYS2 仓库不存在，正确名为
   `mingw-w64-clang-aarch64-winpthreads`（复数带 s）。
   实际上该包是 `mingw-w64-clang-aarch64-clang` 的传递依赖，可直接删掉只留 clang。
